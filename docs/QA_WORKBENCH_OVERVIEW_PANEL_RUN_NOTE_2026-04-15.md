## Design notes

- **Resilient to missing files.** Every section wraps its reads in try/except and emits `(not yet available -- expected at <path>)` instead of crashing. A lane that has not shipped yet simply shows as pending, not a render error.
- **Same-code dual surface.** The GUI tab and the standalone `.md` output come from the same `render_overview_markdown()` function. No drift possible between what management sees in the GUI and what ships in the file.
- **Read-only.** The panel never writes to the repo data directory and never mutates any of the artifacts it reads. The only write in the whole module is `write_overview_markdown()` which targets `HYBRIDRAG_LOCAL_ONLY` by default.
- **Manifest shape handling.** Hardtail run manifests come in two shapes (comparator-written vs CoPilot+-Max supplement). The `_hardtail_timing()` helper reads the comparator manifest first and falls back to `manifest.json` in the same dir when timings are missing. This is how the CoPilot+ row populates even though the canonical comparator manifest has no `timings` key.
- **Production eval shape handling.** The production_eval_results files use `pass_count / partial_count / miss_count / total_queries` at the top level and `per_persona / per_query_type` with nested `{PASS, PARTIAL, MISS, total}` payloads. The renderer reads both shapes and falls back to a nested `benchmark_summary` block if the top-level keys are missing.

## How to refresh after a new run

Option A — GUI: open the eval GUI, the Overview tab is the default, click **Refresh**. Click **Save as .md** to overwrite the on-disk sibling with the current state.

Option B — CLI: from the repo,

```
.venv\Scripts\python.exe -c "from src.gui.eval_panels.overview_panel import write_overview_markdown; print(write_overview_markdown())"
```

That writes `{USER_HOME}\HYBRIDRAG_LOCAL_ONLY\QA_WORKBENCH_OVERVIEW_2026-04-15.md` from current disk state.
