# V1 Harvest Notes

Date: 2026-04-20

Purpose:
- capture V1 ideas worth reviewing for the V2 user-facing panel
- avoid losing subtle UX wins
- stop the team from re-copying clutter by accident

## Review Areas

### Keep Under Review
- scroll bars and narrow-window usability
- hover affordances
- theme switching or light/dark option
- larger or more forgiving question input behavior
- source browsing ergonomics

### Likely Move To Admin Surface
- endpoint selection
- retrieval tuning controls
- Top-K and reranker knobs
- operator diagnostics

### Likely Reject For Default Surface
- any knob that creates cognitive clutter for normal users
- debug-facing labels that distract from asking/reading
- dense control bars that compete with the answer area

## Current Memory Items
- preserve right-side scrolling so the window can shrink without losing access
- consider making the question bar larger for longer questions
- keep V1 subtle UX wins in mind even if not in the immediate slice
- dark/light mode remains a valid later-phase idea, not a current blocker

## Harvest Method
For each V1 idea, classify it as:
- keep on default user surface
- move to admin drawer
- hold for later
- reject

Do not move features forward only because they existed in V1.
Move them forward only if they improve usability in V2.
